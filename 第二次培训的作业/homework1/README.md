## 📚 学习历程记录

### 为什么选择模拟退火算法？

在几种经典启发式算法（遗传算法、蚁群算法、粒子群算法、模拟退火）中，我选择了模拟退火，原因有三：

1. **原理直观**：SA 的物理背景清晰——"高温时粒子活跃可跳出局部最优，低温时趋于稳定收敛"。Metropolis 准则的数学表达 $P = e^{-\Delta E/T}$ 简洁优雅。
2. **超参数少**：主要只需调节初始温度 $T_0$、降温系数 $\alpha$ 和终止温度 $T_{end}$，而 GA 需要设计交叉、变异、选择等多个算子。
3. **代码结构清晰**：SA 的核心只有双重循环（外循环降温 + 内循环产生新解），非常适合作为入门第一个启发式算法。

### 阅读代码过程中的困难与解决

| 困难 | 解决方式 |
|------|---------|
| 不理解为什么对角线距离设为无穷大 | 想到路径不能停留在原地，设为 `inf` 可以自动排除无效路径 |
| 对 `eval_fire` 中 `max(1e-4, exp(-dc/T))` 的下限保护不理解 | 查阅资料得知：当温度极低且差解很大时，`exp(-dc/T)` 可能趋近于 0，导致算法完全无法接受差解而陷入局部最优。保底概率 $10^{-4}$ 给算法保留了极小的"逃生通道"。 |
| 发现 `sa()` 方法中既有退火又有"种群"概念，与标准 SA 不同 | 分析后认为这是 SA + GA 的混合设计：内循环产生 size=100 个新解构成"种群"，从种群中选出最优解，类似于 GA 中每一代的精英保留策略。详见代码注释。 |

## 📊 运行结果分析

### 1. 路径优化效果

| 指标 | 数值 |
|------|------|
| 初始路径长度（随机） | 接近4000 |
| 最终优化路径长度 | 500左右 |
| 优化幅度 | 约 80% |

### 2. 收敛曲线分析

从收敛曲线图可以观察到：

- **前期（T 较高，迭代前 50% 轮次）**：路径长度快速下降（呈现阶梯式），因为此时接受概率高，算法在大范围搜索，容易找到更优解。
- **后期（T 较低，迭代 50%-100% 轮次）**：曲线趋于平坦，算法几乎只接受更优解，最终收敛到（近似）全局最优。

### 3. 参数影响猜想

| 参数 | 增大影响 | 减小影响 |
|------|---------|---------|
| $T_0$（初始温度） | 搜索更充分，收敛更慢 | 可能搜索不充分，过早收敛 |
| $\alpha$（降温系数） | 降温慢，搜索更充分但耗时 | 降温快，可能陷入局部最优 |
| size（种群大小） | 每轮探索更广，但计算量增大 | 计算快，但可能遗漏好解 |
