# 锥桶图像分类项目

## 一、项目简介

本项目使用锥桶图像分类数据集，构建了一个轻量级的卷积神经网络（CNN），实现对 **blue（蓝色）、red（红色）、yellow（黄色）** 三类锥桶图像的自动分类。

数据集规模：
- 训练集（train）：1298 张
- 测试集1（test1）：217 张（用于常规验证）
- 测试集2（test2）：140 张（用于验证模型泛化能力）

## 二、代码流程与细节

### 1. 数据预处理

- **训练集**：使用 `transforms.Compose` 组合了以下数据增强策略，提升模型泛化能力：
  - `Resize([64, 64])`：统一图像尺寸为 64×64（与网络输入尺寸一致）
  - `RandomHorizontalFlip(p=0.5)`：随机水平翻转
  - `RandomRotation(degrees=15)`：随机旋转 ±15 度
  - `ColorJitter`：随机调整亮度、对比度、饱和度
  （1.增加数据多样性，使得模型的鲁棒性更强）
  - `Normalize((0.5, 0.5, 0.5), (0.5, 0.5, 0.5))`：标准化（1.加速收敛，2.缓解激活函数梯度消失）

- **测试集**：由于数据增强有随机性，测试集上仅进行 Resize 和 Normalize，不做数据增强，保证评估的一致性。

### 2. 网络模型设计（`net.py`）

网络名称为 `mixed_net`，整体采用 **4 层卷积 + 4 层池化 + 2 层全连接** 的结构，总计 **10 层**（不含激活函数、BatchNorm、Dropout、Flatten），全连接层为 **2 层**（满足 ≤3 层的要求）。

各层输入输出尺寸如下：

| 层号 | 类型 | 输入尺寸 | 输出尺寸 |
|:---:|:---:|:---:|:---:|
| 1 | Conv2d | [B, 3, 64, 64] | [B, 32, 64, 64] |
| 2 | MaxPool2d | [B, 32, 64, 64] | [B, 32, 32, 32] |
| 3 | Conv2d | [B, 32, 32, 32] | [B, 64, 32, 32] |
| 4 | MaxPool2d | [B, 64, 32, 32] | [B, 64, 16, 16] |
| 5 | Conv2d | [B, 64, 16, 16] | [B, 128, 16, 16] |
| 6 | MaxPool2d | [B, 128, 16, 16] | [B, 128, 8, 8] |
| 7 | Conv2d | [B, 128, 8, 8] | [B, 256, 8, 8] |
| 8 | MaxPool2d | [B, 256, 8, 8] | [B, 256, 4, 4] |
| 9 | GlobalAvgPool2d | [B, 256, 4, 4] | [B, 256, 1, 1] |
| 10 | Linear | [B, 256] | [B, 3] |

- **BatchNorm**：每个卷积层后都加入了 `BatchNorm2d`进行标准化，加速收敛并稳定训练。
- **ReLU 激活**：在每层卷积和第一个全连接后使用，引入非线性。

### 3. 训练流程

1. **设备选择**：自动优先使用 `cuda`，其次 `mps`，最后 `cpu`，我的电脑暂时只有CPU。
2. **损失函数**：`CrossEntropyLoss`（适合多分类任务）。
3. **优化器**：`SGD`（带动量 momentum=0.9，weight_decay=5e-4）。
4. **学习率调度**：`StepLR`，每 30 个 epoch 学习率乘以 0.5，运用了上一次作业当中模拟退火的思想，帮助模型更好地收敛，防止后期震荡。

#### 模型保存

- 训练过程中自动保存 **test1 准确率最高** 的模型到 `pth/` 目录。
- 训练结束后保存最终模型 `model_final.pth`。

### 4. 测试流程（`test.py`）

- 加载训练好的模型权重。
- 分别对 test1 和 test2 进行推理。
- 统计并打印每一类的正确率以及整体正确率。
- 默认优先加载 `pth/model_best_100.00.pth`，若不存在则自动回退到 `model_final.pth`。

## 三、创新点

1. **数据增强策略**：训练时引入随机翻转、旋转、颜色抖动，有效提升模型在不同光照、角度下的泛化能力。
2. **全局平均池化层**：替代第9层的全连接层，以减少参数量，缓解过拟合。
3. **学习率衰减**：运用上次作业学到的模拟退火思想，使用 `StepLR` 动态调整学习率，避免后期震荡，有助于收敛到更优解。

## 六、代码优化历程
### 一、舍弃早停方案
我的代码写完**第一遍**后，发现要训练完200个epoch，而在120多的epoch下，Accuracy就已经收敛，因此增加了一个Patience=20的早停。添加了早停方案后运行**第二遍**发现早停的导致欠拟合，而且最终停下时的精确度有点依赖于随机性。而我构建的神经网络结构过拟合的风险极低，且轻量化训练时长短，因此没有早停的必要性。所以设置Patience=epoch=200，从而舍弃早停。
### 二、用全局平均池化层替换隐藏层当中的全连接层（第9层原来是一个全连接层，但后来改为全局平均池化层），以减少参数量，缓解过拟合：
想要提高模型的泛化能力，即在测试集上的准确率更高，我采用了全局平均池化层，替代全连接层，理由如下：
1. 在双全连接层方案中，要先将矩阵flatten到1维向量，再进行全连接层，且两个全连接层用Dropout(0.5)随机丢弃一半防止过拟合。
2. 在全局平均池化层当中就直接可以池化压缩为一维，有平移不变性。
代码运行**第三遍**，得到最终满意的结果。

## 结果分析
- 1~5 epoch 下，准确率迅速提升至50%以上
- 20~30 epoch 下，准确率稳定提升至90%以上
- 30~70 epoch 下，准确率缓慢提升至95%以上
- 70~80 epoch 下，准确率在96%~98%徘徊
- 83 epoch 出现test1准确率达100%
- 100~200 epoch 下，test1准确率收敛于98%~100%，test2准确率收敛于96%~98%，前期偶然有随机跌落（幸好没有早停），后期基本稳定于最优值（test1在99.5%以上，test2在97.86%）

## 七、关于超参数
- 其实初始超参数batch=1024和epoch=200刚刚好能稳定收敛。
- 这个batch不得不吐槽基本上是一次性把train全部喂过去了，收敛较慢，后面我用batch=256，60多轮就就找到了最优解，然而其到100轮以后才收敛，60~100时虽然大部分是最优解但很不稳定，所以训练效率其实也没什么变化。另外把test2的准确率也提到了98%以上，但test1没有到100%，所以其实最终结果差不多。考虑运气成分和差异的微小性，对于这个项目，影响不算特别大。
- 然后是这个epoch，我觉得100就差不多够了，但是还是运气问题200确实是一个更稳一点的数字
- 综上，超参数调整的必要性较低。
